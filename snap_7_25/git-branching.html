5 Types of git Branching

Master Branch

The Master branch is where all working code exists. We should never develop on the master branch.

Develop Branch

The develop branch serves as the basis for all new code. As each feature branch is tested and complete, it would be added to the develop branch

Feature Branch

This is the branch of code that allows us to devlop small pieces of code and add them into the develop branch. Serves as the basis for most new code. Usually one feature branch per ticket. Once the code is complete, it would be added to the Develop Branch

Release Branch

The Branch that is created just before code is released onto the master branch.
